# Project Astral

Подготовительный пакет для оригинальной кампании Mindustry V8 на JS/Rhino.

## Статус

Проект находится на стадии pre-production. Цель этого репозитория — сначала зафиксировать мир, игровой цикл, экономику и технические границы, затем собрать вертикальный срез без преждевременного производства десятков единиц контента.

Целевая версия игры: **V8 build 159.6**, commit `167be4e`.

## Принципы проекта

1. Начало остаётся понятным опытному игроку Mindustry.
2. Основной цикл постепенно смещается от строительства с нуля к реконфигурации существующей промышленной сети.
3. Поверхность и спутник являются двумя уровнями одной кампании с общим состоянием, прогрессом и логистикой.
4. Старая сеть реагирует на игрока, но только в пределах читаемых и балансируемых правил.
5. Каждый сюжетный сектор вводит цель, изменение системы или важную часть истории.
6. Новая постройка гибкая и дорогая; восстановленный комплекс статичен, рискован, но существенно эффективнее.
7. Финал завершает основной игровой глагол: игрок переподчиняет распределённую сеть, а не просто уничтожает очередное ядро.

## Структура кода

```text
scripts/
  main.js
  core/
  content/
  campaign/
  debug/
bundles/
tools/
```

Сейчас код является безопасным каркасом: он проверяет версию игры, загружает версионируемое состояние кампании и устанавливает базовые события, но намеренно не регистрирует незапроектированный контент.

## Проверка

```bash
python3 tools/validate.py
python3 tools/package.py
```

Готовый архив создаётся в `artifacts/project-astral.zip`.

Каркас проверен встроенным валидатором и реально загружен без JS-ошибок на headless `server-release.jar` V8 build 159.6. Это пока smoke test загрузки, а не тест игрового контента.

## Правило комментариев

В JS-коде используются только комментарии вида `/* ... */`. Однострочные комментарии `//` запрещены валидатором. Комментарии должны объяснять причину решения, ограничение движка или неочевидный контракт, а не пересказывать строку кода.
